# Микро-БЗ для первой линии технической поддержки

Демонстрационный пример базы знаний для первой линии поддержки.

Цель - показать, как можно разложить типовые обращения в понятную структуру: отдельно для клиента, отдельно для оператора, отдельно для эскалаций и обучения новичков.

Это не frontend-проект. Сайт нужен только как удобный способ показать материалы.

## Что внутри

- Клиентские инструкции - короткие тексты, которые можно отправить абоненту в чате.
- Внутренние чек-листы L1 - порядок диагностики для оператора.
- Макросы - готовые формулировки для чата и тикета.
- Правила эскалации - когда передавать обращение дальше.
- Обучающие материалы - базовые термины и примеры для новичка.
- Пример переработки текста - как превратить длинное объяснение в короткую инструкцию.

## Разделы базы знаний

| Раздел | Зачем нужен |
|--------|-------------|
| Клиентские инструкции | Помочь клиенту выполнить простые действия без внутренних терминов. |
| Внутренние чек-листы L1 | Провести оператора по диагностике и не пропустить важные проверки. |
| Макросы и шаблоны | Быстро отвечать клиенту и одинаково фиксировать результат в тикете. |
| Эскалации | Понять, когда проблему нельзя закрыть на первой линии. |
| Обучение новичка | Объяснить базовые сценарии и термины. |
| Структура БЗ | Показать, как можно поддерживать порядок в материалах. |

## Примеры материалов

1. **Как настроить TP-Link EC220 через PPPoE** - клиентская инструкция для чата.
2. **Частые разрывы интернета: L1-чек-лист** - внутренний сценарий диагностики.
3. **Когда оформлять заявку на техника** - правило эскалации.
4. **До/после: переработка клиентской инструкции** - пример редакторской работы.

## Моя роль

- Продумал структуру базы знаний.
- Разделил материалы для клиента и для оператора.
- Написал клиентские инструкции.
- Составил внутренние чек-листы.
- Подготовил макросы для чата и тикетов.
- Описал правила эскалации.
- Отредактировал тексты под разные аудитории.

Оболочка сайта собрана с использованием AI-инструментов и доработана вручную. Она нужна только для просмотра материалов.

Все примеры синтетические и не содержат реальных клиентских данных.

## Как посмотреть

- Открыть `index.html` в браузере.
- Сервер, сборка и зависимости не нужны.
