# Микро-БЗ для первой линии технической поддержки

## О проекте

Анонимизированный портфолио-проект на основе типовых обращений первой линии телеком-поддержки. Цель — показать, как повторяющиеся сценарии поддержки можно превращать в короткие клиентские инструкции, внутренние диагностические чек-листы, макросы для чата, правила эскалации и поддерживаемую структуру базы знаний.

Проект оформлен как работающий статический сайт микро-БЗ, а не набор отдельных документов в Markdown.

## Что демонстрирует проект

- Клиентские инструкции — короткие пошаговые тексты для отправки в чате.
- Внутренние чек-листы L1 — алгоритмы диагностики с ветвлениями решений.
- Макросы и шаблоны для тикетов — готовые формулировки с копированием в один клик.
- Правила эскалации — критерии передачи обращения на L2 или выездному технику.
- Обучение новичка — базовые термины и типовые сценарии для онбординга.
- Архитектура БЗ — структура категорий, типы статей, метаданные, жизненный цикл.
- Метаданные статей — audience, type, status, lastReviewed, relatedArticles.
- Связанные материалы — перекрёстные ссылки между статьями.
- Поиск по базе знаний — по заголовку, описанию, тегам, тексту, типу и аудитории.
- Callout-блоки — визуальные уведомления (warning, info, success) внутри статей.
- Разделение контента — клиентский и внутренний контент разнесены по разным разделам.

## Разделы сайта

| Раздел | Содержание |
|--------|-----------|
| Главная | Все статьи сгруппированы по категориям. |
| Клиентские инструкции | Пошаговые инструкции для абонентов — безопасные, без внутреннего жаргона. |
| Внутренние чек-листы L1 | Алгоритмы диагностики для оператора. Не отправляются клиенту целиком. |
| Макросы и шаблоны | Клиентские макросы и внутренние шаблоны для тикетов с кнопкой копирования. |
| Эскалации | Критерии передачи обращения на техника или сетевую группу. |
| Обучение новичка | Термины, онбординг, примеры переработки материалов. |
| Архитектура БЗ | Структура базы знаний, типы статей, метаданные, lifecycle. |
| О проекте | Описание портфолио-проекта, технологии, как редактировать. |

## Ключевые артефакты

1. **Как настроить TP-Link EC220 через PPPoE** — клиентская чат-инструкция из 9 шагов + блок «Если не получилось». Без admin/admin, без внутренних терминов, с recovery path.
2. **Частые разрывы интернета: L1-чек-лист** — внутренний алгоритм диагностики: 11 шагов с ветвлениями решений (кабель, Wi-Fi, соседи, авария, port-flop).
3. **Структура базы знаний первой линии** — методологическая статья: аудитории, дерево категорий, типы статей, правила именования, метаданные, жизненный цикл.
4. **До/после: переработка клиентской инструкции** — таблица с примерами редактуры: от «стены текста» к коротким инструкциям для чата.

## Как запустить

1. Скачать или склонировать репозиторий.
2. Открыть файл `index.html` в браузере (двойной клик или drag-and-drop).
3. Сервер, сборка и установка зависимостей не нужны.

## Безопасность и анонимизация

Проект не содержит:

- клиентских данных;
- реальных логов или скриншотов рабочих систем;
- внутренних регламентов работодателя;
- реальных SLA и сроков реагирования;
- коммерческой информации;
- названий реальных провайдеров, биллинговых систем или внутренних инструментов;
- конфиденциальных материалов любого рода.

Все материалы синтетические и обезличенные. Любые совпадения с реальными процессами — случайны.

## Моя роль в проекте

- Структура базы знаний и логика разделов.
- Тексты статей и сценарии диагностики.
- Клиентские инструкции, макросы и шаблоны.
- Правила эскалации и критерии передачи.
- Редактура и адаптация текстов под разные аудитории (клиент / оператор / техписатель).
- Разделение клиентского и внутреннего контента.
- Метаданные и связанные статьи.

HTML/CSS/JS-реализация выполнена с использованием AI-инструментов и отредактирована вручную. Дизайн-решения приняты на основе исследования документационных систем (GitBook, ReadMe, Stripe Docs, Microsoft Learn, Docusaurus).
